主从读写分离

作用: 缓解服务器压力,备份

  • 1.开启主库的binlog server_id=1 bin-log=mysql-bin

  • 2.主库中配置复制账号 repl_131 :repl_131

    • 3.分配账号权限: 复制权限 
      GRANT REPLICATION SLAVE ON *.* TO 'repl_113'@'%' IDENTIFIED BY PASSWORD '*EEB072FF15AF65BE3C819B01B2F094FA81681A5A'
    • 4.配置从库配置文件 添加中继日志
      1.change master to master_host="192.168.0.113",master_port=3306,master_user="repl_113",             master_password="repl_113",master_log_file="mysql-bin.000005",master_logg_pos=0;
2. relay-log  relay-log-index

    5.启动复制 start slave
    6.查看复制状态 show slave/master status /G;

    主从配置my.cnf参数

    1.relay-log：中继日志为位置和名称
    2.relay-log-index： 记录中继日志的文件
    3.replicate-do-db： 复制主库中指定的数据库
    4.replicate-do-table：复制主库中指定数据库中的表 database.table
    5.replicate-ignore-table 主从中忽略的表
    3.log-slave-updates：从库从主库复制的数据会写入log-bin日志文件里。这也是该参数的功能

    逻辑备份

    mysqldump 备份 相当于导出
    导出: mysqldump -u root -p db table > /www/name.sql
    导入: mysql -u root -p db < /www/name.sql
    mysqldumper 备份工具
    mysqldumper优点
    1.多线程，可以是转存数据快很多
    2.mydumper的输出已于管理和分析，因为他的表和元数据是分开的单独文件。
    导出: mydumper -u root -p root -B database -t table -o /www/name.sql
    导入 myloader -u root -p root -d /www/name.sql

slave的I/O线程接收到数据后，将接收到的日志内容依次写入到relay log文件最末端，并将读取到的主库bin log文件名和位置position记录到master-info文件中，以便在下一次读取用; slave的SQL线程检测到relay log中内容更新后，读取日志并解析成可执行的SQL语句，这样就实现了主从库的数据一致;
