主从复制(docker-redis)

复制分为主节点(master)和从节点(slave) 都是主节点复制到从节点
有一对多和一对一
1.缓解服务器的压力

2.备份

  • 实现过程:

    docker

    首先构建网络来分配给做主从复制redis的ip
    然后run容器 挂载数据卷位置 ip 端口号映射 网络名称
    docker run -itd -v /redis/master:/redis -p 6350:6379 –network=mynetwork
    –ip=192.160.1.150 –name redismaster redis

    redis

    在redis配置文件中关掉保护和绑定Ip

    从节点复制主节点命令 : slaveof masterHost masterPort
    info replication 查看主从信息
    info server 查看runid
    取消复制命令 : slaveof no one

主从复制过程

  • psync 根据id 和偏移量去主服务器里面找

    1、如果没找到id那么主服务器将全量复制 2、如果找到id那么部分复制

  • 复制积压缓冲区 主节点写命令时会复制一份到缓冲区还存储了对应字节的偏移量 (先进先出)所以保存的是主节点最近执行的写命令

  • 他们之间会以心跳来检测是否存在

    1.保存主节点信息
    2.主从建立socket
    3.发送ping命令
    4.权限验证
    5.同步数据集
    6.命令持续复制

    读写分离

    • 实现过程: 
1.  在php中配置中配置主节点(master)和从节点(slave)的ip(宿主机Ip)和端口(运行容器时映射的端口)
2.  进行主节点和从节点的连接
3.  随机分配哪一个从节点进行读  负载均衡（在有多个从节点的情况下）
4.  配置写命令(主节点)和读命令(从节点)并判断使用的命令是否在自己的配置数组命令中再分配是主节点实现还是从节点实现
