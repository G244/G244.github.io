laravel 请求周期

1.创建一个应用程序/服务容器 ?
2.kernel Http内核 继承了父类,里面的bootstrappers数组定义了需要执行的任务
3.kernel http内核 经过定义的htpp中间件
4.kernel handle 方法相当一一个大盒子 request请求 response响应
### 服务提供者 (整个周期最重要的组成部分)
  所有的服务提供者均配置到config/app.php providers 数组中 所有register方法注册后才会调用boot方法 开启了多种的组件 路由,服务容器绑定,中间件
6.请求调度:request会传递给路由调度请求->中间件->控制器
7.几乎所有的服务容器绑定都在服务提供者中注册 
8.延时服务提供者
  直到注册绑定的服务真的需要时才会加载   
  要延迟加载提供者，需要实现 \Illuminate\Contracts\Support\DeferrableProvider 接口并       置一个 provides 方法。这个 provides 方法返回该提供者注册的服务容器绑定：
9.facades为服务容器提供了静态接口,底层类的静态代理 __callStatic代理调用
10.laravel的契约是一组接口

laravel csrf

csrf跨站请求伪造: 让一个用户去点击已经信任过的网址跳转到指定的这个网站手动改过的请求
 1.调用界面是通过在session中储存_token值,然后@csrf是获取session的_token值,再把cookie XSRF-TOKEN生成 
 1).表单请求的时候根据前端传的**_token参数**传递值跟服务端做校验 
 2).通过头部**X-CSRF-TOKEN**传递值跟服务端校验 
 3）根据头部**X-XSRF-TOKEN**传递获取cookie中的XSRF-TOKEN值到服务端中框架进行解密成session的_token值再做校验
