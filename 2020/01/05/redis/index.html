Redis

Redis 是完全开源免费的,遵守BSD协议，是一个高性能(NOSQL)的key_value数据库

key-value 缓存产品的特点：
1.Redis支持数据的持久化，可以将内存中的数据保存到磁盘中
2.同时还提供list,set,zset,hash等数据结构的存储
3.支持数据的备份(master-slave) ,集群等高可用功能（支持16个库）
4.消息队列（聊天室）
5.单线程，原子性

redis类型

1.string类型是二进制安全的 可以包含任何数据
二进制安全特点:编码，解码发生在客户端完成，执行效率高,不需要频繁编解码,不会出现乱码
2.hash类型存储值对象(2^32-1)40多亿个,储存一个用户对象数据
3.list类型可以存入多个值（适合）
  ~~对数据量大的集合数据删减
  列表数据显示(分页)、关注列表、粉丝列表、留言评价、热点新闻（top5）
  ~~任务队列
  订单系统下单流程、用户系统登录注册短信
 4.set类型 是唯一，无序集合
  共同关注、共同喜好、二度好友 唯一性统计访问网站的独立IP
 5.zset类型，是唯一，有序集合
  排行榜，权重的队列(任务计划)

BSD协议开源协议是给使用者很大的自由协议

string hash list set Zset hyper log log

redis订阅发布

即时聊天、群聊
粉丝关注你，你发消息推送消息给粉丝

Redis 事务

1.执行命令错误不会进行回滚,则是报出错的信息，其他命令依旧执行
2. 如果是命令语法错误,执行后则进行回滚
 商品秒杀、转账活动

Redis 数据淘汰策略

aof-rewrite-incremental-fsync  ，默认为 32MB 写入数据量
maxmermory redis最大缓存
maxmemory-policy noeviction 根据应用场景,选择淘汰策略
LRU存储  如果redis中存在则写入的时候先删除这个键值再去存入一遍(知道这个键值是热数据)


Redis 持久化

RDB机制

RDB: 记录数据
dump.rdb 以快照的方式写入到二进制文件🀄
快照保存数据快，还原数据快
RDB文件结构
大写的为常量是存储在这两个里面保持不变的 
|REDIS|db_version|databases|EOF|check_sum
REDIS:告诉redis我是持久化文件
db_version 备份的版本
databases 备份数据
EOF:标识当前回复结束了
check_sum:校验字符 
databases
database 0 | database 2
SELECTDB | db_number | key_value_pairs
一个字节       第几个库           实际存的数据

RDB手动持久化

save 阻塞当前redis服务器,直到RDB过程完成为止
bgsave redis进程执行fork操作创建子进程,RDB持久化过程由于进程负责,完成后自动结束，阻塞只发生在fork阶段

RDB的优缺点

优点:在恢复数据集的时候速度比AOF更快    
缺点:没有办法做到实时持久化/秒级持久化

AOF持久化

AOF:记录命令  
aof文件会越来越大 （进行重写（多条命令进行合并，过期命令去掉,5.0以上会把文件转换为二进制文件））
appendonly yes aof开启 
bgrewriteaof  重写
  • appendfsync 存入方式
  aof always 每写一次命令就存进aof文件中(不经过缓冲区) 
  aof everysec 每秒存入文件中(经过缓冲区)
  aof no 从不刷写，直到内核缓冲区满了时，才会刷写到磁盘
当命令存入AOF缓冲区并根据方式存入硬盘，因为子进程只能共享fork操作时的内存数据,由于父进程依然在响应命令.redis使用重写缓冲区保存这部分新数据
当重写缓冲区生成新的AOF文件就信号通知父进程再新的AOF文件替换旧的AOF的文件
  • AOF写入文件中的命令示例
*2 标识有2个操作占位符
$6 标识 "SELECT" 的字符串长度
SELECT
$1 标识 "0" 的字符串长度
0

额外命令

info Persistence 持久化状态
持久化分割命令:redis-cli infp Persistence | grep rdb_bgsave_in_progress | awk -F ':' '{print $2}'
检查持久化文件信息
redis-check-aof file.aof 
redis-check-rdb file.rdb
